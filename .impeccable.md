## Design Context

### Users
- Primary users are older adults recording speech and daily state, plus family members and project researchers reviewing progress.
- Typical context is low-friction, trust-first browsing on desktop and mobile, often in guided project/demo settings.
- Core job: understand platform value quickly, browse speech/evaluation data clearly, and feel safe using the workflow.

### Brand Personality
- Warm, trustworthy, calm.
- Emotional goal: reduce anxiety, communicate care and reliability, avoid flashy or aggressive visual language.
- Voice should feel human and respectful, with clear information density for non-technical audiences.

### Aesthetic Direction
- Dual-theme strategy: keep a deep night mode for focus and presentation; add a bright day mode for readability and long-duration viewing.
- Visual tone: lifestyle-premium, minimal and refined, restrained palette, strong hierarchy, sparse accents.
- Current signature accent is warm amber/gold; this should stay the single dominant accent across themes.
- Theme switch is manual user control (not auto by time).
- Day mode should simplify texture and decorative noise compared to night mode.

### Design Principles
- Prioritize legibility over decoration, especially for data-heavy sections.
- Use one primary accent color to establish hierarchy quickly; avoid multi-accent competition.
- Keep structure stable across themes so mode switching changes tone, not information architecture.
- Ensure interactive affordances are obvious without requiring instructions.
- Design for composure: generous spacing, clean surfaces, and predictable visual rhythm.
